1. Introduction to Tails OS and Core Components

Tails OS (The Amnesic Incognito Live System) is a security-focused, Debian-based Linux distribution designed for privacy and anonymity. It routes all internet connections through the Tor network and leaves no trace on the computer by default. Tails includes a suite of privacy-enhancing applications and tools, making it ideal for journalists, activists, whistleblowers, and anyone needing strong digital privacy.

2. Application Overviews, Histories, Case Studies, and Getting Started Guides

Debian

History:

Founded in 1993 by Ian Murdock, Debian is a free, open-source operating system developed collaboratively. It introduced advanced package management and a strong commitment to free software principles. Debian is the foundation for many distributions, including Ubuntu and Tails OS.

Case Study:

A university research lab adopted Debian for its computational biology cluster. Debian’s stability and extensive software repositories allowed researchers to install and update scientific tools efficiently, supporting reproducible research and minimizing system downtime.

Getting Started:
  1. Download the Debian ISO from debian.org.
  2. Create a bootable USB drive using Rufus or Balena Etcher.
  3. Boot the computer from the USB drive.
  4. Follow the installer prompts for language, location, and keyboard.
  5. Partition the disk as needed.
  6. Set up a user account and password.
  7. Choose software to install.
  8. Complete installation and reboot.

GNOME

History:

Launched in 1997 by Miguel de Icaza and Federico Mena Quintero, GNOME is a free, user-friendly desktop environment for Linux. GNOME 3 introduced a modern, streamlined interface. It is governed by the GNOME Foundation and is widely used as a default desktop environment.

Case Study:

A European government agency migrated thousands of workstations to Linux with GNOME. The intuitive interface and accessibility features reduced training time and supported employees with disabilities, while centralized management improved security and reduced costs.

Getting Started:
  1. Install a Linux distribution that includes GNOME (e.g., Debian, Fedora).
  2. Log in; GNOME loads as the default desktop.
  3. Explore the Activities Overview (Super/Windows key).
  4. Access applications from the grid or search bar.
  5. Use Settings to customize appearance and shortcuts.
  6. Install GNOME extensions for added functionality.

Tails OS

History:

First released in 2009, Tails OS was created for privacy and anonymity. Built on Debian, it routes all internet traffic through Tor and leaves no trace on the host computer. It is widely used by journalists, activists, and privacy advocates.

Case Study:

An investigative journalist in a restrictive country used Tails OS from a USB stick to research sensitive topics and communicate securely. Tails’ amnesic design and Tor integration protected the journalist’s identity and data, even under surveillance.

Getting Started:
  1. Download the Tails ISO from tails.boum.org.
  2. Create a bootable USB stick using the Tails Installer or Etcher.
  3. Boot the computer from the USB stick.
  4. Configure language, keyboard, and settings at the Welcome Screen.
  5. Start Tails; all internet traffic is routed through Tor.
  6. Use Persistent Storage (optional) for encrypted files/settings.

Kleopatra (on Tails OS)

History:

Kleopatra is part of the Gpg4win suite, providing a graphical interface for cryptographic operations. It supports OpenPGP and X.509 standards and was integrated into Tails OS to make encryption accessible to users needing strong privacy protections.

Case Study:

A human rights activist used Kleopatra on Tails OS to generate and manage cryptographic keys, encrypt emails, and verify signatures. This ensured confidential and authentic communication with colleagues in high-risk environments.

Getting Started:
  1. Boot into Tails OS.
  2. Open Kleopatra from the Applications menu.
  3. To create a new key pair:
    • File > New Certificate
    • Enter name/email, set a passphrase, complete wizard
    • Back up key pair
  4. To import a public key:
    • File > Import Certificates
    • Select key file
  5. To encrypt a file:
    • Right-click file, select Sign/Encrypt, choose recipient’s public key
  6. To decrypt a file:
    • Right-click encrypted file, select Decrypt/Verify

KeePassXC

History:

KeePassXC is a community-driven fork of KeePassX, itself based on the original KeePass for Windows. It offers a modern, cross-platform password manager with strong encryption and is actively maintained by the open-source community.

Case Study:

A small business used KeePassXC to manage employee credentials. The encrypted database and password generator improved security, while access controls ensured only authorized staff could view sensitive information.

Getting Started:
  1. Install KeePassXC from your distribution’s package manager or keepassxc.org.
  2. Launch KeePassXC.
  3. Click “New Database” to create a password database.
  4. Set a strong master password (and optionally a key file).
  5. Add entries for each account.
  6. Use the password generator for strong, unique passwords.
  7. Save the database file securely.
  8. Use search and autofill features as needed.

Electrum (on Tails OS)

History:

Electrum was created in 2011 by Thomas Voegtlin as a lightweight, secure Bitcoin wallet. It uses a client-server protocol, allowing users to manage Bitcoin without downloading the entire blockchain. Electrum is open-source and is included in Tails OS.

Case Study:

A freelance journalist used Electrum on Tails OS to receive payments in Bitcoin. By running Electrum in Tails, the journalist ensured that wallet data and transaction history were not stored on any device after use. Electrum’s support for hardware wallets and encrypted backups provided additional security.

Getting Started:
  1. Boot into Tails OS.
  2. Open Electrum from the Applications menu.
  3. Create a new wallet or restore an existing one.
  4. Write down and securely store the recovery seed phrase.
  5. Set a strong password for the wallet.
  6. Receive Bitcoin by sharing your wallet address.
  7. Send Bitcoin by entering the recipient’s address and amount.
  8. Use the “History” tab to view transactions.
  9. Backup your wallet file and seed phrase securely.

VeraCrypt (on Tails OS)

History:

VeraCrypt was launched in 2013 as a fork of the discontinued TrueCrypt project. Developed by Mounir Idrassi, VeraCrypt addressed security vulnerabilities found in TrueCrypt and introduced stronger encryption algorithms and improved security features. VeraCrypt is widely used for creating encrypted containers and encrypting entire drives, and is included in Tails OS.

Case Study:

A traveling consultant used VeraCrypt on Tails OS to store sensitive client documents. By creating an encrypted container, the consultant ensured that even if the USB drive was lost or stolen, the data remained inaccessible without the correct password.

Getting Started:
  1. Boot into Tails OS.
  2. Open VeraCrypt from the Applications menu.
  3. Click “Create Volume” to start the wizard.
  4. Select “Create an encrypted file container.”
  5. Choose a location and name for the container file.
  6. Select encryption and hash algorithms.
  7. Specify the size of the container.
  8. Set a strong password.
  9. Format the container.
  10. To use:
    • Open VeraCrypt, select a slot, click “Select File,” choose your container
    • Click “Mount,” enter your password
    • Access the mounted volume from the file manager
    • When finished, click “Dismount”

GNU Image Manipulation Program (GIMP)

History:

GIMP was first released in 1996 by Spencer Kimball and Peter Mattis as a free and open-source raster graphics editor. Developed under the GNU Project, GIMP has grown into a powerful alternative to proprietary image editing software, supporting advanced features such as layers, masks, filters, and extensibility through plugins and scripts.

Case Study:

A nonprofit organization used GIMP to create promotional materials, edit photographs, and design social media graphics. By leveraging GIMP’s advanced features and free licensing, the organization saved on software costs and empowered volunteers to contribute creative work from any platform.

Getting Started:
  1. Install GIMP from your distribution’s package manager or gimp.org.
  2. Launch GIMP.
  3. Open an image file or create a new canvas via File > Open or File > New.
  4. Use the toolbox for selection, painting, and editing tools.
  5. Manage layers in the Layers panel.
  6. Apply filters and effects from the Filters menu.
  7. Export your work via File > Export As.
  8. Explore plugins and scripts for additional functionality.

Pidgin

History:

Pidgin is a free, open-source instant messaging client that supports multiple chat protocols through a plugin system. Originally released as Gaim in 1998, it was renamed Pidgin in 2007. Pidgin is included in Tails OS with the Off-the-Record (OTR) plugin for encrypted messaging.

Case Study:

A group of activists used Pidgin with OTR encryption on Tails OS to coordinate protests and share sensitive information. By using anonymous accounts and encrypted chats, they minimized the risk of surveillance and interception.

Getting Started:
  1. Boot into Tails OS.
  2. Open Pidgin from the Applications menu.
  3. Add a new account (e.g., XMPP/Jabber, IRC) using anonymous credentials.
  4. Enable the Off-the-Record (OTR) plugin via Tools > Plugins.
  5. Start a conversation and click the OTR button to initiate encrypted chat.
  6. Verify contacts using OTR’s authentication methods.
  7. Avoid saving chat logs and always use anonymous accounts for sensitive communication.

Thunderbird

History:

Thunderbird is a free, open-source email client developed by the Mozilla Foundation. First released in 2003, Thunderbird supports multiple email protocols and can be extended with add-ons for encryption and privacy. On Tails OS, Thunderbird is pre-configured to work with Enigmail for OpenPGP email encryption.

Case Study:

A whistleblower used Thunderbird on Tails OS to communicate securely with journalists. By encrypting emails with OpenPGP and using anonymous email accounts, the whistleblower protected their identity and the confidentiality of their messages.

Getting Started:
  1. Boot into Tails OS.
  2. Open Thunderbird from the Applications menu.
  3. Set up a new email account using anonymous credentials.
  4. Configure OpenPGP encryption (Enigmail is integrated by default):
    • Generate or import an OpenPGP key pair
    • Associate the key with the email account
  5. Exchange public keys with contacts for encrypted communication.
  6. Compose and send encrypted emails by selecting the encryption option before sending.
  7. Regularly verify and update trusted keys, and avoid using personal information in email headers or signatures.

3. Creating a New User on Tails OS with Encryption

Tails OS does not support traditional multi-user accounts, but you can create a secure, personalized environment for a new user by leveraging Persistent Storage, encrypted directories, and Kleopatra for encryption.

  1. Boot Tails OS and configure language/keyboard at the Welcome Screen.
  2. Set Up Persistent Storage:
    • Applications > Tails > Configure Persistent Storage
    • Set a strong passphrase
    • Enable features (Personal Data, GnuPG, KeePassXC, etc.)
    • Restart Tails to activate
  3. Create a User Folder:
    • In Persistent Storage, create /home/amnesia/Persistent/username
  4. Configure Application Settings:
    • Customize browser bookmarks, email, and app preferences
    • Save configs in the user’s folder
  5. Set Up Encryption with Kleopatra:
    • Open Kleopatra
    • File > New Certificate, enter user info, set passphrase
    • Back up key pair in user’s folder
    • Export public key for sharing
    • Import trusted contacts’ public keys
  6. Encrypt/Decrypt Files:
    • Right-click file > Sign/Encrypt (choose recipient’s key)
    • Right-click encrypted file > Decrypt/Verify
  7. Secure Communication:
    • Encrypt messages/files before sending
    • Decrypt received files/messages with Kleopatra
  8. Best Practices:
    • Back up keys and passphrases offline
    • Use strong, unique passphrases
    • Store passwords in KeePassXC
    • Regularly update trusted keys

4. Maximizing Privacy Using All Apps and Settings on Tails OS

System and Session Settings

  • Boot Tails from trusted hardware; verify image signature.
  • Use the Welcome Screen for minimal, non-identifying configuration.
  • Enable Persistent Storage only for essential data; use a strong passphrase.
  • Always use the latest Tails version; restart for each session.

Network and Internet Privacy

  • All traffic is routed through Tor by default; never bypass Tor.
  • Use Onion Services (.onion) when possible.
  • Disable unnecessary network protocols (Bluetooth, Wi-Fi).
  • Use bridges/pluggable transports if Tor is censored.

Application-Specific Privacy

  • Tor Browser:
    • Set Security Level to “Safest”
    • Disable JavaScript/plugins
    • Use New Identity/Circuit features
    • Clear cookies/site data after each session
    • Never open downloads directly
  • Kleopatra:
    • Generate strong OpenPGP keys
    • Store private keys only in Persistent Storage
    • Verify fingerprints before trusting keys
    • Encrypt/sign all sensitive files and emails
  • KeePassXC:
    • Store all credentials in an encrypted database
    • Use strong, unique passwords
    • Lock database when not in use
    • Back up database offline
  • Electrum:
    • Use a strong wallet passphrase
    • Store wallet file in Persistent Storage, optionally encrypt with VeraCrypt
    • Never reuse wallet addresses
    • Avoid linking wallet to personal identity
  • VeraCrypt:
    • Encrypt all sensitive files
    • Use strong algorithms and passwords
    • Dismount volumes when not in use
  • GIMP:
    • Strip metadata before sharing images
    • Edit sensitive images only within Tails
    • Store images in encrypted containers
  • Pidgin:
    • Use anonymous accounts
    • Enable OTR for all chats
    • Verify contacts
    • Avoid saving chat logs
  • Thunderbird:
    • Use anonymous email accounts
    • Encrypt all emails with OpenPGP
    • Regularly verify/update trusted keys
    • Avoid personal info in headers/signatures

General Best Practices

  • Never reveal personal information in any app or online service.
  • Use Tails for all sensitive activities; avoid mixing with non-anonymous sessions.
  • Shut down Tails after each session to erase all traces.
  • Regularly review and update all encryption keys and passwords.
  • Educate yourself on operational security (OpSec) principles.
